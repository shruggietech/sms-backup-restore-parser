## Purpose and Context

<div style="text-align:justify">

Three bugs were discovered during the first real-world test of the v0.1.0 release binary. This sprint fixes all three, expands the test suite to cover the gaps that allowed them to ship, and cuts a v0.1.1 hotfix tag.

</div>

**Bug 1 — Multi-file `--combined` produces separate files instead of one merged output.** When the user passes multiple XML files (e.g., a calls export and an SMS export) with `--combined`, the CLI loops over each input file independently and calls `parse_backup()` per-file. Each call writes its own `<stem>.json` combined file. The result is two separate JSON files, each with empty arrays for record types not found in that particular input, instead of one merged JSON containing records from all inputs.

**Bug 2 — PyInstaller binary does not bundle `jsonschema`.** The `[validate]` optional dependency is never installed into the build environment, and no `--hidden-import` directive is passed to PyInstaller. The resulting binary always fails on `--validate` with an `ImportError` telling the user to pip-install the extra. Release binaries must be full-featured; "lite" builds are never acceptable for official releases unless explicitly labeled as such.

**Bug 3 — `__main__.py` does not propagate exit codes.** The entry point calls `main()` but discards its return value instead of passing it to `sys.exit()`. The CLI always exits with code 0 regardless of errors. This bug is documented in `test_cli.py` comments but was never fixed.

<div style="text-align:justify">

In addition to these three bugs, the test suite has critical infrastructure gaps. Test fixtures (XML files under `tests/fixtures/`) are caught by the blanket `*.xml` pattern in `.gitignore` and were never committed to the repository. A clean clone cannot run any tests. Beyond that, there are zero tests for multi-file parsing, zero tests for the validate module, zero tests for the writer classes in isolation, and zero tests for compact output mode. This sprint addresses all of these.

</div>

#### D. Fix multi-file `--combined` to produce a single merged output

The root cause is in `cmd_parse()` in `src/sms_backup_parser/cli.py`. The function loops over each input file and calls `parse_backup()` independently. When `--combined` is used with multiple inputs, the correct behavior is to merge all records from all inputs into a single combined JSON file.

**D.1.** Add a new function `parse_backup_multi` to `src/sms_backup_parser/parser.py`, after the existing `_parse_combined` function at the end of the file:

```python
def parse_backup_multi(input_paths, output_dir, strip_media=False,
                       inject_date_iso=True, indent=2, progress_tracker_factory=None):
    """Parse multiple XML files into a single combined JSON output.

    Merges all SMS, MMS, and call records from all input files into one
    combined JSON file with top-level 'sms', 'mms', and 'calls' arrays.

    Args:
        input_paths: List of Path objects to XML backup files.
        output_dir: Directory for the combined output file.
        strip_media: If True, omit base64 'data' attribute from MMS parts.
        inject_date_iso: If True, add computed date_iso fields.
        indent: JSON indentation level (None for compact).
        progress_tracker_factory: Optional callable returning a ProgressTracker.
            Called once per input file.

    Returns:
        dict with keys 'sms_count', 'mms_count', 'call_count', 'output_files'.
    """
    output_dir = Path(output_dir)
    output_dir.mkdir(parents=True, exist_ok=True)

    stem = Path(input_paths[0]).stem
    output_path = output_dir / f"{stem}.json"
    total_counts = {"sms": 0, "mms": 0, "call": 0}

    with CombinedJsonWriter(output_path, indent=indent) as writer:
        for input_path in input_paths:
            input_path = Path(input_path)
            if not input_path.exists():
                raise FileNotFoundError(f"Input file not found: {input_path}")
            if not input_path.is_file():
                raise FileNotFoundError(f"Not a file: {input_path}")
            if input_path.stat().st_size == 0:
                raise ValueError(f"Input file is empty: {input_path}")

            tracker = progress_tracker_factory() if progress_tracker_factory else None
            if tracker:
                tracker.start()

            try:
                context = ET.iterparse(str(input_path), events=('end',))
                for event, elem in context:
                    tag = elem.tag

                    if tag == 'sms':
                        record = _extract_sms(elem, inject_date_iso)
                        writer.write_record("sms", record)
                        total_counts["sms"] += 1
                        if tracker:
                            if tracker._verbosity >= 2:
                                tracker.update_verbose("sms", record)
                            else:
                                tracker.update("sms")
                        elem.clear()

                    elif tag == 'mms':
                        record = _extract_mms(elem, inject_date_iso, strip_media)
                        writer.write_record("mms", record)
                        total_counts["mms"] += 1
                        if tracker:
                            if tracker._verbosity >= 2:
                                tracker.update_verbose("mms", record)
                            else:
                                tracker.update("mms")
                        elem.clear()

                    elif tag == 'call':
                        record = _extract_call(elem, inject_date_iso)
                        writer.write_record("calls", record)
                        total_counts["call"] += 1
                        if tracker:
                            if tracker._verbosity >= 2:
                                tracker.update_verbose("call", record)
                            else:
                                tracker.update("call")
                        elem.clear()

            except ET.ParseError as e:
                raise ET.ParseError(
                    f"Malformed XML in {input_path}: {e}"
                ) from e

            if tracker:
                tracker.finish()

    output_files = [str(output_path)] if any(total_counts.values()) else []

    return {
        "sms_count": total_counts["sms"],
        "mms_count": total_counts["mms"],
        "call_count": total_counts["call"],
        "output_files": output_files,
    }
```
