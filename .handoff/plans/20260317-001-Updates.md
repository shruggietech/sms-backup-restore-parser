## Purpose and Context

<div style="text-align:justify">

This sprint brings the sms-backup-restore-parser repository into conformance with ShruggieTech documentation and specification conventions established by shruggie-indexer and metadexer. The primary deliverable is the revised technical specification (`reference/knowledge-base.md`), which has been expanded from a flat XML format reference into a properly structured specification covering the full project scope. Secondary deliverables align the MkDocs documentation site styling, add missing docs pages, synchronize docs content with the specification, and add the docs dependency group to `pyproject.toml`.

</div>

---

## Implementation Ordering

<div style="text-align:justify">

Sections 1 and 2 have no dependencies and may be executed in parallel. Section 3 depends on Section 2 (the mkdocs.yml must be updated before new pages are added). Section 4 depends on Section 3 (docs content updates reference the new nav structure). Section 5 (specification and changelog update) is the final section and is gated on all preceding work being verified.

</div>

### Description

<div style="text-align:justify">

Add new navigation entries for the XML Format Reference page and Changelog page.

</div>

  name: material
  palette:
    scheme: slate
  features:
    - navigation.tabs
    - navigation.instant
    - search.highlight

  - Home: index.md
  - Installation: installation.md
  - Usage: usage.md
  - Schema Reference: schema.md
  - XML Format Reference: xml-reference.md
  - Reports: reports.md
  - Changelog: changelog.md
